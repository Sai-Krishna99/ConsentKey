# ConsentKey Demo Script (60s, 2 Speakers)

## 0-6s: Positioning Slide (What / Why)
**Voice**
- **Sai:** AI coding agents move fast.
- **Ankit:** Production systems are fragile.
- **Sai:** ConsentKey is a physical consent firewall for high-risk AI actions.
- **Ankit:** It inserts deliberate human intent before execution.

**Tone**
Calm, matter-of-fact.

**Visual**
- Show your existing ConsentKey slide:
  - `Physical consent firewall for high-risk AI actions`
  - `AI coding agents move fast.`
  - `Production systems are fragile.`
  - `ConsentKey inserts deliberate human intent.`

**Direction**
Keep this to ~6 seconds max. If possible, reveal the three body lines one by one.

## 6-12s: Trigger Risky Action
**Voice**
- **Sai:** Here, the agent proposes a high-risk production restart.
- **Ankit:** This is exactly the kind of action that should pause for human review.

**Visual**
- Operator Mode ON
- Trigger HIGH scenario
- Operator Mode OFF so the console fills the frame

## 12-20s: Interception (Hero Moment)
**Voice**
- **Sai:** ConsentKey intercepts the request before execution.
- **Ankit:** On the MX Creative Console and Actions Ring, it surfaces the exact command, risk level, and blast radius in one view.

**Visual**
- Pending state visible
- Risk badge
- Haptic indicator
- Diff preview
- Blast radius line

**Direction**
Pause briefly so this state lands.

## 20-27s: Deny Path
**Voice (firmer)**
- **Sai:** No approval, no execution.
- **Ankit:** The agent is blocked until consent is explicitly granted.

**Visual**
- Click `Deny`
- Show `Blocked - Consent Required`
- Terminal shows `403 Consent Required`

**Direction**
This is the first mic-drop moment.

## 27-37s: Approve Path
**Voice**
- **Sai:** Now we trigger the same high-risk action again.
- **Ankit:** For higher-risk actions, ConsentKey requires deliberate physical confirmation through the console workflow.

**Visual**
- Trigger HIGH again
- Hold to approve
- Show hold progress
- Show token issued
- Show terminal update

**Direction**
Keep pacing smooth and intentional.

## 37-50s: Architecture Page (How It Works)
**Voice**
- **Sai:** Under the hood, ConsentKey sits between the AI agent and the execution layer.
- **Ankit:** It classifies risk, pauses unsafe commands, and only issues a short-lived token after verified approval through MX Creative Console and Actions Ring.

**Visual**
- Show your existing architecture page
- Pan/zoom (if needed) to highlight:
  - `AI Agent`
  - `Risk / Policy layer`
  - `MX Creative Console + Actions Ring`
  - `Short-lived token`
  - `Execution layer`

**Direction**
Do not explain every box. Use this section as a credibility pass, not a deep dive.

## 50-60s: Close (Impact)
**Voice**
- **Sai:** ConsentKey keeps AI agents useful in production without removing human control.
- **Ankit:** Today it runs through MX Creative Console and Actions Ring, and the same trust layer can extend to MX Master 4 next.

**Visual**
- End slide: `ConsentKey`
- Tagline: `Verified human intent for AI actions`
- Hold on approved state / logo for about 1 second before cut
